# 📌 Parking Lot

> 以下想法暂不纳入当前版本，后续视社区反馈再决定是否推进。

## 1. 繁简/异体字对照

在 schema 中新增 `variants` 字段，列出碑文中出现的异体字及其对应的通行字。对古文字研究者有价值，但初版会显著增加贡献者门槛。

## 2. 原碑行列矩阵

为每座碑增加 `matrix` 字段，按行列存储碑文（保持原碑竖排格式，从右到左）。便于还原碑文原貌和学术研究，但工作量大，仅纯文本释文的数倍。

## 3. 同一碑帖多版本释文

在 `data/stele/{dynasty}/{name}/` 目录下存放多个 JSON（依据不同版本/考释）。初版只收一个通行版。

## 4. 全文检索索引

用 SQLite / ElasticSearch / Meilisearch 为所有碑帖建立全文检索。可作为独立的下游项目（例如 companion 项目）。

## 5. 碑帖图片链接 & 对照查看

在 metadata 中加 `image_url` 字段，引用博物馆公开的高清图像链接。但图片版权和链接稳定性需单独评估。

## 6. 白话翻译 / 注释

为碑文加白话翻译或词句注释。这是另一个独立产品。

## 7. CLI 查询命令行工具

提供一个 `stele search <keyword>` 命令，离线检索本地碑帖数据库的释文。非常有价值，可作为独立项目。

## 8. Web 查询 UI

简单的静态 Web 页面，输入关键字查碑帖全文。同上。

## 9. 避讳字自动检测

自动检测碑文中可能的避讳字（如唐碑避"民"讳）。需要完整的避讳字数据库，初版可由人工在 `description` 字段标注。

## 10. 书法体识别

为 OCR 识别或用户提供书法书体自动分类功能（篆/隶/楷/行/草）。依赖机器学习模型，超出数据仓库范围。
